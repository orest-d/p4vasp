  25  """ 
  26  Store module handles configurable xml persistance - i.e. it makes possible 
  27  to store and retrieve python object into (from) xml files. 
  28  It is possible to customize the encoding of objects as well as to select xml tags 
  29  or attributes that are used for storing a particular object. 
  30  To store objects causing difficulties (e.g. python code) marshal and pickle 
  31  encoded in a hexadecimal form are used. 
  32   
  33  For convenience the functions *load*, *loads*, *dump*, *dumps* are defined, 
  34  that can be used the same way as the *load*, *loads*, *dump*, *dumps* in the *pickle* 
  35  module. 
  36   
  37  Here is a more advanced example: 
  38   
  39    # Create example class 
  40    class Test: 
  41      def __init__(self,a=0): 
  42        self.a=a 
  43        self.b=2 
  44        self.c=2.2 
  45        self.d=[4,5] 
  46        self.e=type(int) 
  47      def __str__(self): 
  48        return "Test(a=%s,b=%s,c=%s,d=%s,e=%s)"%( 
  49        str(self.a),str(self.b),str(self.c),str(self.d),str(self.e)) 
  50      __repr__=__str__ 
  51   
  52    # Create list 
  53    l=[1,2,3] 
  54    # Now there will be circular references: 
  55    l.append(Test(l)) 
  56   
  57    # Create the top-level store profile 
  58    profile=Profile() 
  59   
  60    # Create the store-profile for class Test: 
  61    testprofile=Profile(Test,tagname="Test",attr_setup=''' 
  62    a 
  63    int   B b 
  64    float c 
  65    list d 
  66    ''') 
  67   
  68    # Using addAttr and AttributeProfiles attributes can be configured 
  69    # more precisely: 
  70    testprofile.addAttr(HexPickleAttribute("e",tagattr=1)) 
  71   
  72    # Register the testprofile in the top-level store profile 
  73    profile.addClass(testprofile) 
  74   
  75    # Now dump ane load l using the profile: 
  76    print l 
  77    s = profile.dumps(l) 
  78    print s 
  79    print profile.loads(s)
